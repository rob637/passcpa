Three frameworks, three different purposes. Let's sort them out so you never confuse them again. <break time='0.8s'/> COBIT. Think: Control Objectives. COBIT is about IT governance and management. It helps organizations ensure that IT delivers value and manages risk. When the exam asks about aligning IT with business goals, or about governance frameworks, COBIT is usually the answer. <break time='0.5s'/> Memory trick: COBIT has Objectives. Big picture governance. <break time='1s'/> ITIL. Think: IT Service Management. ITIL is about delivering IT services efficiently. Incident management, problem management, change management, service desk. When the exam asks about IT operations and service delivery, ITIL is usually the answer. <break time='0.5s'/> Memory trick: ITIL serves IT. Service management. <break time='1s'/> ISO 27001 and 27002. Think: Information Security. ISO 27001 is the standard for information security management systems. 27002 provides the security controls. When the exam asks about security governance or security standards, ISO 27000 series is usually the answer. <break time='0.5s'/> Memory trick: ISO secures Info. Security standards. <break time='0.8s'/> Quick summary to lock it in. COBIT governs. ITIL serves. ISO secures. <break time='0.5s'/> One more tip: these frameworks aren't competitors. Organizations often use all three together. COBIT for governance, ITIL for operations, ISO for security. They're complementary layers.
