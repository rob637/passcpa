# CIA Part 2: Practice of Internal Auditing Cheat Sheet
## IIA Exam - Performing the Work

---

## Domain I: Managing the Internal Audit Activity (20%)

### Planning the IA Activity
| Element | Description |
|---------|-------------|
| Risk-Based Plan | Prioritizes based on organizational risk |
| Audit Universe | All auditable units/activities |
| Resource Allocation | Staff, time, budget per engagement |
| Coordination | With external auditors, other providers |

### Audit Plan Development
1. Understand organizational objectives
2. Develop/update audit universe
3. Perform risk assessment
4. Prioritize audits
5. Estimate resources
6. Communicate plan to board

### CAE Responsibilities
| Area | Responsibility |
|------|----------------|
| Strategy | Align IA with organization goals |
| Resources | Ensure adequate staffing/skills |
| Quality | Maintain QAIP |
| Relationships | Board access, management communication |
| Reporting | Results to board and management |

### Coordination with Other Assurance
- External auditors
- Regulators
- Other assurance providers (risk, compliance)
- Minimize duplication
- Ensure coverage

## Domain II: Planning the Engagement (20%)

### Engagement Planning Steps
1. Establish objectives
2. Define scope
3. Allocate resources
4. Develop work program
5. Identify evaluation criteria

### Preliminary Survey Activities
| Activity | Purpose |
|----------|---------|
| Review prior audits | Learn history |
| Gather background | Understand area |
| Review policies | Know requirements |
| Identify risks | Focus work |
| Interview management | Get perspectives |

### Engagement Objectives
| Type | Focus |
|------|-------|
| Assurance | Evaluate control design and effectiveness |
| Consulting | Provide advice and recommendations |

### Setting Scope
- What to include and exclude
- Time period covered
- Locations/systems
- Limitations and constraints

### Work Program Elements
| Component | Content |
|-----------|---------|
| Objectives | What to achieve |
| Procedures | Steps to perform |
| Resources | Who and when |
| Timeline | Milestones and deadlines |

### Risk Assessment for Engagement
- Inherent risk of the area
- Control environment factors
- Prior issues and findings
- Complexity and materiality

## Domain III: Performing the Engagement (40%)

### Evidence Requirements (RASC+U)
| Criterion | Description |
|-----------|-------------|
| Reliable | Trustworthy source |
| Relevant | Relates to objective |
| Sufficient | Adequate quantity |
| Competent | Quality appropriate |
| Useful | Supports conclusions |

### Evidence Types (Strongest to Weakest)
1. Documentary - External (bank confirmations)
2. Documentary - Internal (signed policies)
3. Physical - Observation/inspection
4. Testimonial - Interviews, inquiries

### Audit Procedures
| Technique | Description |
|-----------|-------------|
| Inquiry | Asking questions |
| Observation | Watching processes |
| Inspection | Examining documents/assets |
| Confirmation | Third-party verification |
| Reperformance | Re-executing procedures |
| Analytical | Comparing, trending, ratio analysis |

### Sampling Methods
| Type | Use |
|------|-----|
| Statistical | Projects to population |
| Judgmental | Targets specific items |
| Attribute | Testing controls (yes/no) |
| Variable | Testing amounts |

### Analytical Procedures
| Type | Example |
|------|---------|
| Trend Analysis | Compare over time |
| Ratio Analysis | Compare relationships |
| Reasonableness | Expectations vs. actual |
| Benchmarking | Compare to peers/standards |

### Working Papers Requirements
| Characteristic | Description |
|----------------|-------------|
| Complete | Support conclusions |
| Clear | Understandable |
| Organized | Logically structured |
| Referenced | Cross-referenced |
| Reviewed | Supervisory sign-off |

### Working Paper Contents
| Element | Purpose |
|---------|---------|
| Objective | What the work addresses |
| Scope | What was tested |
| Results | What was found |
| Conclusion | Auditor's assessment |
| Preparer/Date | Accountability |
| Review | Supervisory approval |

## Domain IV: Communicating Results and Monitoring (20%)

### Communication Criteria (7 Cs)
| Criterion | Description |
|-----------|-------------|
| Accurate | Factually correct |
| Objective | Unbiased, balanced |
| Clear | Easy to understand |
| Concise | No unnecessary detail |
| Constructive | Focus on improvement |
| Complete | All relevant info |
| Timely | Issued promptly |

### Report Components
| Section | Content |
|---------|---------|
| Objective | Purpose of engagement |
| Scope | What was reviewed |
| Findings | Condition, criteria, cause, effect |
| Recommendations | Actionable improvements |
| Management Response | Planned actions |
| Overall Opinion | If applicable |

### Finding Elements (4 Cs)
| Element | Description |
|---------|-------------|
| Condition | What exists (the problem) |
| Criteria | What should exist (the standard) |
| Cause | Why the gap exists |
| Consequence | Impact or risk |

### Report Distribution
- CAE determines recipients
- Based on content sensitivity
- Consider who needs to know
- Follow organizational policy

### Follow-Up Process
1. Track open items
2. Request status updates
3. Verify corrective action
4. Assess effectiveness
5. Report to management/board
6. Escalate if unresolved

### Monitoring Resolution
| Status | Action |
|--------|--------|
| Open | Track progress |
| In Progress | Verify milestones |
| Implemented | Test effectiveness |
| Risk Accepted | Document and report |
| Escalated | Report to board |

## Key Formulas & Metrics

### Sample Size Considerations
Higher sample if:
- Higher confidence needed
- Higher expected error rate
- Lower tolerable error

### Audit Time Analysis
**Actual Hours ÷ Budgeted Hours = Efficiency Rate**

### Findings Metrics
**Implementation Rate = Closed Findings ÷ Total Findings**

## Quick Memory Aids

### Evidence Hierarchy
**CORA** - Confirmation, Observation, Recalculation, Analytical

### Report Quality
**ACCCCTC** - Accurate, Clear, Concise, Constructive, Complete, Timely, Objective

### Sampling Decision Tree
- Need statistical precision? → Statistical sampling
- Targeting specific risks? → Judgmental sampling
- Control testing? → Attribute sampling
- Amount testing? → Variable sampling
